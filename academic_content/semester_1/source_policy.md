# Academic source policy

## Source priority

1. Approved SDU course descriptions and official learning outcomes.
2. Primary scientific literature and official software documentation when method details or interfaces may change.
3. Established graduate and advanced undergraduate textbooks.
4. Previous university material used only for prerequisite reinforcement and diagnostic context.
5. Original synthetic examples created specifically for this project.

## Core source registry

### Official curriculum

- SDU semester-one descriptions for DM857, DM847, BMB830, and BMB831.

### Programming and computational reasoning

- John V. Guttag, *Introduction to Computation and Programming Using Python*, third edition.
- Allen B. Downey, *Think Python*, third edition.
- Existing validated DM857 content in this repository.

### Bioinformatics

- Current official documentation for databases, formats, and tools used in examples.

- Kevin P. Murphy, *Probabilistic Machine Learning: An Introduction*.
- Probability, biostatistics, linear-model, and ANOVA material supplied from prior university study.

### Computational biomedicine context

## Copyright constraints

The repository contains original educational synthesis. Sources may guide:

- factual accuracy;
- topic sequence;
- terminology;
- standard equations and algorithms;
- creation of new examples and questions.

The repository must not contain:

- copied textbook chapters or substantial passages;
- reproduced proprietary figures or tables;
- copied exercise sets or answer keys;
- close paraphrases that merely substitute words;
- embedded copies of copyrighted books.

Source maps identify intellectual foundations but do not imply copied wording.

## Freshness policy

Software APIs, package behavior, databases, recommended workflows, and biological resources may change. Content must distinguish:

- stable concepts and mathematics;
- version-sensitive software behavior;
- current database interfaces;
- illustrative workflows versus normative protocols.

Before freezing a release, version-sensitive claims should be checked against official documentation and recorded with an access date or version.

## Biomedical evidence policy

- Synthetic datasets must be labelled as synthetic.
- A teaching example must not be described as a validated clinical workflow.
- Association, prediction, mechanism, and causation must remain distinct.
- Statistical significance must not be presented as biological or clinical importance.
- Limitations, uncertainty, and plausible alternative explanations must be explicit.
- Patient-level recommendations are outside the scope of these course materials.

## Citation granularity

Each module must include a source map linking source IDs to:

- official outcome alignment;
- factual foundations;
- algorithmic or statistical references;
- software documentation;
- biological context.

Individual memory cards need source anchors when they contain version-sensitive, unusually specific, or potentially disputed claims. Standard definitions and original calculations may inherit the module-level source map.

## Review requirements

Before publication, each module must pass:

- scientific accuracy review;
- prerequisite and progression review;
- language-parity review;
- example and calculation verification;
- answer-key and distractor review;
- memory-card duplication review;
- copyright and provenance review;
- limitation and biomedical-restraint review.
